## рҹҢҢ What is MissionControl?

<table>
<tr>
<td width="52%">

**MissionControl** is a high-performance, standalone **Volunteer Logistics Command Center** вҖ” built exclusively for food rescue operations. Originally an embedded module inside the **Nourish Connect** platform, it has been fully decoupled into its own independent, serverless application.

The design philosophy draws from **Tesla's tactical interfaces** and modern **aerospace command centers** вҖ” dark-mode-first aesthetics, glassmorphism UI panels, real-time data streams, and fluid Framer Motion animations throughout.

> **This is not a form. This is not a CRUD app.**  
> **This is a mission-critical operations terminal.**

**Who uses MissionControl?**

| Role | What they do |
|------|-------------|
| рҹ§‘вҖҚрҹӨқвҖҚрҹ§‘ **Volunteer Couriers** | Accept, track, and complete food rescue missions |
| рҹҸў **Food Bank Dispatchers** | Monitor active missions, trigger emergency alerts |
| рҹ“Ҡ **Operations Leads** | Review XP data, rank progression, mission analytics |

The Emergency Alert HUD is a **full-surface interruption overlay** that activates when:
- рҹ•җ A food consignment reaches the **90-minute expiry threshold**
- рҹҸӢпёҸ Volume is classified as **HIGH** вҖ” 100+ meals equivalent
- рҹ“Қ At least one qualified volunteer is within the operational zone

**Alert Lifecycle:**
```
Firestore trigger fires (expiry_threshold crossed)
            вҶ“

            вҶ“

            вҶ“
Volunteer taps  [ ACCEPT EMERGENCY MISSION ]
            вҶ“
HUD dismisses with Framer Motion exit animation
            вҶ“
Mission auto-loads in /tracking/[alertId]
```

// lib/utils/xp.ts
export function calculateXP(mission: Mission, volunteer: Volunteer): number {
  let xp = 50; // base completion reward

  // Urgency bonus
  if (mission.urgency === "CRITICAL") xp += 75;
  if (mission.urgency === "HIGH")     xp += 25;

  // Load size bonus
  if (mission.weightKg >= 50) xp += 30;

  // Daily streak multiplier
  const multiplier = volunteer.streak >= 3 ? 1.5 : 1.0;

  return Math.floor(xp * multiplier);
}

**Step 3 вҖ” Paste Security Rules**

In **Firestore вҶ’ Rules**, replace the default rules with:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    match /volunteers/{userId} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId;
    }

    match /missions/{missionId} {
      allow read: if request.auth != null;
      allow update: if request.auth != null
        && resource.data.status == "AVAILABLE"
        && request.resource.data.claimedBy == request.auth.uid;
    }

    match /deliveries/{deliveryId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null
        && resource.data.volunteerId == request.auth.uid;
    }

    match /alerts/{alertId} {
      allow read: if request.auth != null;
      allow write: if false; // Dispatchers only вҖ” use Admin SDK
    }
  }
}
```

## рҹӣ пёҸ Troubleshooting

<details>
<summary><b>вқҢ "Missing or insufficient permissions" вҖ” Firestore</b></summary>

1. Ensure the user is authenticated before any Firestore access
2. Check your **Firestore вҶ’ Rules** match the exact paths your code reads/writes
3. Use the Firebase Emulator to debug locally:
   ```bash
   firebase emulators:start
   ```

</details>

<details>
<summary><b>вқҢ "window is not defined" вҖ” Map not rendering</b></summary>

You're importing a Leaflet component without `ssr: false`. Fix:

```typescript
// вң… Correct вҖ” SSR-safe
const TrackingMap = dynamic(() => import("./TrackingMap"), { ssr: false });

// вқҢ Wrong вҖ” will crash on Next.js server render
import TrackingMap from "./TrackingMap";
```

</details>

<details>
<summary><b>вқҢ Peer dependency conflict on install (React 19)</b></summary>

```bash
npm install --legacy-peer-deps
# or add to .npmrc:
echo "legacy-peer-deps=true" >> .npmrc && pnpm install
```

</details>

<details>
<summary><b>вқҢ Framer Motion animations not working</b></summary>

All Framer Motion components must be in Client Components:

```typescript
"use client"; // вҶҗ Required at the top of the file

import { motion } from "framer-motion";
```

</details>

<details>
<summary><b>вқҢ GPS position not updating on the map</b></summary>

1. Check browser has granted **location permission** for your origin
2. `navigator.geolocation` requires HTTPS вҖ” `localhost` is the only exception
3. Verify Firestore `deliveries` write rules permit the current user

</details>
